How it works

One tool call. A real shopping trip.

Bench is an A2MCP service. An agent calls one tool with the competing endpoints; Bench does the shopping and returns a ranked scorecard with evidence.

1

Probe each service

Reachability, MCP handshake, and a valid x402 paywall — Bench confirms each candidate can actually be bought from before spending anything.

2

Buy from every one

Bench makes one real, settled purchase from each service and records the price paid, the response latency, and how much it actually delivered.

compare_services(targets, paid_tool)
3

Rank by value

A transparent score — price 45%, latency 25%, delivery 30% — crowns the best value and explains, in plain language, why each service placed where it did.

Provably real

Real purchases. Real receipts.

Every Bench verdict is backed by an on-chain settlement, not a simulation. A broken service that takes payment and delivers nothing is caught and excluded — it can never win on price alone.

Settlement runs on public infrastructure with the payer holding a bounded, capped wallet. Same inputs, same verdict, every time.
